## 上传长文 workflow

1. 确保当前标签页已由 OpenClaw Browser Relay 扩展附着，然后导航到长文发布页：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay navigate "https://creator.xiaohongshu.com/publish/publish?source=official&from=tab_switch&target=article"`
2. 等待页面稳定：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay wait --time 3000`
3. 获取交互快照：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay snapshot --efficient --interactive --compact`
4. 如果页面先停在“新的创作 / 导入链接 / 长文合集”，点击“新的创作”：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay click <新的创作ref>`
5. 等待编辑器加载并重新快照：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay wait --time 3000`，然后 `/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay snapshot --efficient --interactive --compact`
6. 输入标题：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay type <标题输入框ref> "{标题}"`
7. 优先尝试写入正文：
   - 短正文可直接输入：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay type <正文输入框ref> "{正文内容}"`
   - 长正文先复制到剪贴板：`printf '%s' "{正文内容}" | pbcopy`
   - 聚焦正文编辑区后粘贴：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay click <正文编辑区ref>`，再 `/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay press "Meta+V"`
8. 如果执行 `type` / `Meta+V` 后正文仍为空、字数没变，使用长文编辑器 fallback：
   - `/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay evaluate --fn '() => { const el = document.querySelector(".tiptap.ProseMirror"); if (!el) return { ok: false, reason: "no-editor" }; el.innerHTML = "<p>{正文内容}</p>"; el.dispatchEvent(new InputEvent("beforeinput", { bubbles: true, cancelable: true, inputType: "insertText", data: "{正文内容}" })); el.dispatchEvent(new InputEvent("input", { bubbles: true, inputType: "insertText", data: "{正文内容}" })); el.dispatchEvent(new Event("change", { bubbles: true })); el.dispatchEvent(new KeyboardEvent("keyup", { bubbles: true, key: "1" })); return { ok: true, text: el.innerText }; }'`
9. 如果正文里要插图，先把图片复制到剪贴板，再回到编辑区粘贴：
   - `osascript -e 'set the clipboard to (read (POSIX file "/tmp/openclaw/uploads/<图片文件>") as «class PNGf»)'`
   - `/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay press "Meta+V"`
10. 编辑器底部红色主按钮有时会显示“一键排版”，但它就是当前阶段的主前进按钮。正文和标题确认无误后点击它：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay click <主前进按钮ref>`
11. 等待并重新快照。如果页面进入封面模板/封面设置阶段，并出现明确的“下一步”按钮，则点击：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay click <下一步ref>`
12. 等待并重新快照：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay wait --time 2000`，然后 `/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay snapshot --efficient --interactive --compact`
13. 如果进入最终发布设置页后页面提示“笔记图片生成中，请稍后...”，不要立即点击发布，先等待该提示消失。
14. 如果页面有简介/描述框且任务给了简介，输入简介：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay type <简介输入框ref> "{简介内容}"`
15. 如果有话题，输入 `#{话题}` 后执行：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay press Enter`
16. 发布前重新快照：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay snapshot --efficient --interactive --compact`
17. 点击发布按钮：`/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay click <发布按钮ref>`
18. 发布后等待并确认：
   - `/Users/openclawcn/.homebrew/bin/openclaw browser --browser-profile chrome-relay wait --time 1500`
   - 优先检查 URL 是否带 `published=true`
   - 或确认页面已返回发布首页 / 笔记管理页

## 注意

- 小红书无图发布默认走本 workflow，不要留在图文上传页等待图片
- 小红书长文编辑器本质是 `ProseMirror`，普通 `type` / `Meta+V` 失败时要允许使用编辑器 fallback
- 页面阶段切换后 ref 会整体变化，进入“新的创作”、封面设置、最终发布设置后都要重新 `snapshot`
- 编辑器底部红色主按钮的文案可能从“一键排版”变成“下一步”，都按“当前阶段主前进按钮”处理
