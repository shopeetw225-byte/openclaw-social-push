## 发布知乎想法 workflow

以下命令遵循 `SKILL.md` 中定义的 `OPENCLAW_BIN` 约定，统一使用 `"$OPENCLAW_BIN"`。

实测记录：`2026-03-23` 已在真实账号 + `chrome-relay` 环境下完成知乎想法真实发布验证。

1. 确保当前标签页已由 OpenClaw Browser Relay 扩展附着，然后导航到知乎首页：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay navigate "https://www.zhihu.com/"`
2. 等待页面初步稳定：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay wait --time 3000`
3. 获取交互快照：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay snapshot --efficient --interactive --compact`
4. 优先点击顶部 `创作` 按钮，再点击菜单里的 `发想法`，进入独立想法发布面板：
    - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay click <创作按钮ref>`
    - 重新 `snapshot`
    - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay click <发想法按钮ref>`
5. 重新 `snapshot`，确认当前页面已经出现独立想法发布面板；如果首页同时残留另一组内嵌创作块，优先操作最后一组、当前可见的 `标题` / 正文 / `发布` 控件
6. 点击标题输入框并清空旧内容：
    - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay click <标题输入框ref>`
    - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay press "Meta+A"`
    - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay press Backspace`
7. 输入标题：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay type <标题输入框ref> "{标题}"`
8. 点击正文输入框并输入纯文本内容：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay type <正文输入框ref> "{想法内容}"`
9. 如 `发布` 仍不可用，先等待一次前端校验；必要时让标题和正文输入框失焦后重新 `snapshot`
10. 重新快照确认发布按钮可用：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay snapshot --efficient --interactive --compact`
11. 点击发布按钮：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay click <发布按钮ref>`
12. 发布后等待并确认：
    - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay wait --time 1500`
    - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay snapshot --efficient --interactive --compact`
    - 优先确认 1-2 个明确成功信号：出现 `发布中` 后进入成功状态、发布浮层已关闭、或后续请求中出现 `POST /api/v4/content/publish` 返回 `200`

## 注意事项

- 如果快照出现“登录 / 注册”，说明当前浏览器未登录知乎，先登录再继续
- 页面状态预期：想法 workflow 应停留在 `www.zhihu.com` 的想法发布器；如果进入文章编辑器（`zhuanlan.zhihu.com`），说明当前不在想法发布态，需返回想法入口
- 实测更稳的入口是顶部 `创作 -> 发想法` 菜单，而不是只依赖首页内嵌创作块
- 在独立发想法面板里，标题与正文填完后，可能需要等待一次前端校验或让输入框失焦，`发布` 按钮才会从禁用变为可用
- 如果页面同时出现两组 `标题` / 正文 / `发布` 控件，优先操作最后一组、且与独立发想法面板绑定的那组
- 首阶段仅覆盖纯文本想法发布与确认，不承诺图片、视频、回答行为
- 回答 flow 在本阶段故意不实现（只处理文章与想法）
- 视频 / 付费专栏 / 直播能力均超出本阶段范围
