## 发布知乎文章 workflow

以下命令遵循 `SKILL.md` 中定义的 `OPENCLAW_BIN` 约定，统一使用 `"$OPENCLAW_BIN"`。

实测记录：`2026-03-23` 已在真实账号 + `chrome-relay` 环境下完成知乎文章真实发布验证。

1. 确保当前标签页已由 OpenClaw Browser Relay 扩展附着，然后导航到知乎文章编辑页：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay navigate "https://zhuanlan.zhihu.com/write"`
2. 等待页面初步稳定：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay wait --time 3000`
3. 获取交互快照：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay snapshot --efficient --interactive --compact`
4. 点击标题输入框并清空旧内容：
   - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay click <标题输入框ref>`
   - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay press "Meta+A"`
   - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay press Backspace`
5. 输入标题：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay type <标题输入框ref> "{标题}"`
6. 准备正文（长文本优先走剪贴板）：
   - `pbcopy < "{正文文件路径}"`
   - 或 `printf '%s' "{正文内容}" | pbcopy`
7. 点击正文编辑区并粘贴：
   - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay click <正文编辑区ref>`
   - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay press "Meta+V"`
8. 等待编辑器状态稳定：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay wait --time 1500`
9. 重新快照并确认可发布：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay snapshot --efficient --interactive --compact`
10. 点击发布按钮：`"$OPENCLAW_BIN" browser --browser-profile chrome-relay click <发布按钮ref>`
11. 发布后等待并确认：
    - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay wait --time 1500`
    - `"$OPENCLAW_BIN" browser --browser-profile chrome-relay snapshot --efficient --interactive --compact`
    - 优先确认 1-2 个明确成功信号：出现发布成功提示，或页面跳转到文章详情页 / URL 不再停留在编辑器

## 注意事项

- 如果快照出现“登录 / 注册”，说明当前浏览器未登录知乎，先登录再继续
- 页面状态预期：文章 workflow 应停留在 `zhuanlan.zhihu.com` 的文章编辑器；如果看到首页想法发布器，说明当前不在文章编辑页，需回到文章路径重试
- `chrome-relay` 实测在单页连续操作下更稳定；如果多标签切换后出现 `tab not found` / `fetch failed`，优先只保留当前文章页并重新附着扩展
- 首阶段仅覆盖标题 + 正文 + 发布确认，不承诺行内图片上传、封面上传、视频、回答、付费专栏行为
- 回答 flow 在本阶段故意不实现（只处理文章与想法）
- 视频 / 付费专栏 / 直播能力均超出本阶段范围
